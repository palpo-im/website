# Contribution Guide

We warmly welcome contributions to this project! Before you begin, please take a moment to read this guide to ensure your contributions proceed smoothly and align with project standards.

## Code of Conduct

We are committed to fostering an open and inclusive environment for all contributors. Please adhere to our Code of Conduct.

## How to Contribute

### Reporting Bugs

If you discover any bugs, please submit them via [GitHub Issues](https://github.com/palpo-im/palpo/issues).

When filing a bug report, please provide as much detail as possible, including:

*   Steps to reproduce
*   Expected behavior
*   Actual behavior
*   Error messages (if any)
*   Your environment information (operating system, version, etc.)

### Submitting Feature Requests

If you have ideas for new features, please also submit them via [GitHub Issues](https://github.com/palpo-im/palpo/issues).

When submitting a feature request, describe your idea and explain how it will improve the project.

### Submitting Code

1.  **Fork the Project**: First, fork this project to your GitHub account.
2.  **Clone the Repository**: Clone the forked repository to your local machine.
    ```bash
    git clone https://github.com/your-username/palpo-im.git
    cd palpo-im
    ```
3.  **Create a Branch**: Create a new branch for your feature or bug fix.
    ```bash
    git checkout -b feature/your-feature-name
    # or
    git checkout -b bugfix/your-bug-fix-name
    ```
4.  **Make Changes**: Implement your code changes on your branch.
5.  **Write Tests**: If your changes involve new features or bug fixes, please write corresponding tests.
6.  **Run Tests**: Ensure all tests pass.
7.  **Commit Changes**: Commit your changes with a clear commit message.
    ```bash
    git add .
    git commit -m "feat: Add your feature" # or "fix: Fix your bug"
    ```
8.  **Push to Remote Repository**: Push your branch to your forked repository.
    ```bash
    git push origin feature/your-feature-name
    ```
9.  **Create a Pull Request**: Create a Pull Request on GitHub to merge your changes into the `main` branch of the main repository.

## Code Style

Please follow the existing code style of the project. We use Prettier for code formatting. You can run it before submitting your code:

```bash
bun prettier --write .
```
